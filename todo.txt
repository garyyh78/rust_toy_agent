[1] dispatch_tool_calls now takes &Mutex<TodoManager> instead of Arc<Mutex<TodoManager>>
    [DONE]
    Files: src/agent_loop.rs, src/tools.rs, src/e2e_test.rs, src/bin_core/test_mode.rs,
           src/bin_core/state.rs
    Why it matters: holding an Arc::new() per call was wasteful; the State holds a
                    plain Mutex<TodoManager>. Using &Mutex avoids the per-call allocation
                    and clarifies ownership. The std::sync::Mutex guard is dropped before
                    any .await, so there is no deadlock risk.

    Changes made:
      - Changed dispatch_tool_calls signature from &Arc<Mutex<TodoManager>> to &Mutex<TodoManager>
      - Changed agent_loop signature from &Arc<Mutex<TodoManager>> to &Mutex<TodoManager>
      - Changed dispatch_tools signature from &Arc<Mutex<TodoManager>> to &Mutex<TodoManager>
      - Updated all callers (e2e_test, test_mode, tools tests)
      - Removed Arc import from agent_loop, tools, e2e_test, test_mode

    Acceptance: cargo clippy -- -W clippy::await_holding_lock passes; no
                test deadlocks under --test-threads=1; Arc<Mutex<TodoManager>> no
                longer appears in signatures.

[2] safe_path() uses precomputed WorkdirRoot to avoid repeated canonicalize syscalls [DONE]
    File: src/tool_runners.rs (WorkdirRoot, safe_path)
    Why it matters: canonicalize() does a syscall per path component. The agent
                    calls safe_path dozens of times per loop.

    Changes made:
      - Added WorkdirRoot struct with original + canonical paths computed once
      - Changed all tool runners and dispatch to accept &WorkdirRoot
      - safe_path now uses the precomputed canonical root

    Acceptance: existing tests pass; safe_path no longer calls canonicalize
                on every invocation.

[3] BackgroundManager uses DashMap for tasks and mpsc for notifications [DONE]
    File: src/background_tasks.rs
    Why it matters: avoids global lock contention on every background spawn
                    and check.

    Changes made:
      - Replaced Arc<Mutex<HashMap<...>>> tasks with Arc<DashMap<...>>>
      - Replaced Mutex<Vec<Notification>> notification_queue with mpsc::unbounded_channel
      - drain_notifications now polls the receiver via try_recv()

    Acceptance: background_command_runs_exactly_once and test_drain_notifications
                still pass; no Mutex on the task map or notification queue.

[4] run_write propagates mkdir errors instead of silently ignoring them [DONE]
    File: src/tool_runners.rs (run_write)
    Why it matters: if mkdir fails (permission denied, ENOTDIR), the subsequent
                    fs::write fails with confusing errors.

    Changes made:
      - Changed let _ = std::fs::create_dir_all(parent) to propagate the error

    Acceptance: mkdir errors are now surfaced; test pass.

[5] llm_client::jitter() uses thread-local PRNG instead of clock nanos [DONE]
    File: src/llm_client.rs
    Why it matters: clock-nanos-based jitter was correlated across concurrent
                    retries, defeating the purpose.

    Changes made:
      - Added thread-local RNG (rand::rngs::StdRng) seeded from SystemTime
      - Replaced nanos-based calculation with rng.gen_range()

    Acceptance: rand = "0.8" added to Cargo.toml; jitter produces different
                delays across concurrent calls.
